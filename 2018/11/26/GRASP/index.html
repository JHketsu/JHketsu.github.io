GRASP 相関概念

GRASP

General responsibility assignment Software patterns：通用职责分配软件模式
GRASP模式是面向对象设计（OOD）的一些很好的实践原则，可以提高软件的可维护性、可重用性、理解性。

OOD

Oriented-Object Design defines software objects and how they collaborate to fulfill the requirements.OOD定义软件对象以及它们之间如何协作以满足需求。
确定类的方法，确定对象之间怎么通信。

input for OOD

需求规约、词汇表、用例模型以及其对应的系统顺序图和规约、领域模型

output from OOD

设计模型
交互图、类图、包图等等

职责

Responsibility-Driven Design (RDD)：如何为对象分配职责
职责是关于类目的规约或义务，它可以包括行为、对象创建、数据存储等等。

Doing 行为职责

  1. 自己的行为
  2. 在其他对象中初始化
  3. 与其他对象的控制、合作活动有关

    Knowing 认知职责

  4. 知道自己的private封装数据
  5. 知道相关对象
  6. 知道可以计算或导出的d东西

职责在类和方法中的表现取决于该职责的粒度，有的职责涉及很少的方法，有的确涉及很多。另外，关于认知职责可以从领域模型中推理，毕竟领域模型提供了一些联系。

模式

‘’ In object technology, a pattern is a named description of a problem and solution that can be applied to new contexts . ‘’
GRASP模式描述了对象设计和责任分配的基本原则。
GRASP模式包括
Information Expert 信息专家，
Creator 创建者，
High Cohesion 高内聚，
Low Coupling 低耦合，
Controller 控制器，
Polymorphism 多态，
Indirection 间接，
Pure Fabrication 纯虚构，
Protected Variations 隔离变化。
